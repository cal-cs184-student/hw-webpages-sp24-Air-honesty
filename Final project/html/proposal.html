CS 184/284A: Final project- Leap and Jump

Yifeng Zhang, Sihua Ren, Aslina Cheng, Boyuan Ma

In this project, we want to use texture and Light rendering to make an interactive application based on the existing platform jumping game. In the showcase, users can control the intensity of the jump by controlling how long they click on the screen to get a higher score with beautiful animation and fetching background music.

Problem Description

In the original small game called “tiao-yi-tiao”, it is a platform jumping game with a single color action model and geometric blocks, which was rather boring. We wanted to upgrade the game with a good visual appearance and improve the experience by texture and light rendering. The difficulties we might meet are Real-Time Rendering and Input Lag. Managing real-time light rendering and texture processing while ensuring the game runs smoothly can be challenging, especially on devices with limited processing power. And Minimizing the delay between user input (clicking or tapping the screen) and the application's response is crucial for a good user experience. But we are determined to present it as best we can by searching for information and trying a lot for optimization.

Goals and Deliverables

Baseline Plan:

  1. Scene Design and Display
    • We will construct a virtual scene using hierarchical modeling techniques, comprised of multiple virtual objects.
    • The virtual scene will be displayed and can represent either an indoor or outdoor environment.
    • The scene will include the ground as an essential element.
  2. Texture Mapping
    • We plan to apply texture mapping to at least two main objects within the scene, enhancing their visual detail and realism.
  3. Lighting, Materials, and Shadow Effects
    • To achieve realistic lighting and shadow effects, we will implement the Phong reflection model for rendering.
    • The integration of material properties and shading will contribute to the authenticity of the scene's appearance.
  4. User Interaction and Viewpoint Switching
    • Users will be able to control and transform objects within the scene, such as moving, rotating, and scaling, using keyboard or mouse inputs.
    • We will implement camera transformations that allow for arbitrary angle viewing, providing a comprehensive exploration of the scene.
  5. Experiments and evaluation
    • We will measure the frames per second (FPS) during interaction and scene manipulation to ensure smooth real-time performance. The target is to maintain a minimum of 30 FPS, which is considered acceptable for real-time applications.
    • For static scenes, we'll measure the time taken to render a single frame. Our goal is to keep this rendering time under a threshold that ensures a seamless user experience.

Aspirational Plan

Based on the baseline,through GUI selection, we provide diverse textures and looks, making it more colorful and more fun.
